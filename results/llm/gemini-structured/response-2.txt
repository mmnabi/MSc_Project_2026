### **Cybersecurity Threat Identification Report for NordFinans Bank AB**

**Prepared for:** NordFinans Bank AB Executive Committee
**Prepared by:** Cybersecurity Consultant
**Date:** October 26, 2023
**Framework:** MITRE ATT&CK® for Enterprise v14

This document outlines relevant cyber threats facing NordFinans Bank AB, mapped to the 14 tactics of the MITRE ATT&CK framework. The analysis is based on the provided organizational profile, considering the bank's infrastructure, services, regulatory environment, and known security posture.

---

### **1. Reconnaissance**

Adversaries will actively and passively gather information to plan their attack. NordFinans' public profile as a bank with a significant digital footprint makes it a prime target.

-   **T1595.002 - Active Scanning: Vulnerability Scanning:**
    -   **Relevance:** The bank's web banking portal, mobile app backend, and Open Banking APIs are internet-facing. Adversaries will continuously scan these for known vulnerabilities (e.g., in .NET, Azure App Service, or API gateways). The bank's quarterly external scans by Qualys may not be frequent enough to detect and remediate vulnerabilities before an attacker discovers them.
-   **T1593.001 - Search Open Websites/Domains: Social Media:**
    -   **Relevance:** Adversaries can identify key personnel (e.g., SWIFT operators, database administrators, Azure cloud engineers) through LinkedIn. This information is invaluable for crafting highly targeted spear-phishing campaigns. Job postings on the bank's website or other platforms can also reveal specific technologies in use (e.g., "Temenos T24 Specialist," "Palo Alto Firewall Engineer"), guiding an attacker's tool development.
-   **T1590 - Gather Victim Host Information:**
    -   **Relevance:** By analyzing DNS records, SSL certificates, and network blocks, an adversary can map NordFinans' hybrid infrastructure. They can identify the use of Azure Front Door, Microsoft 365, and other cloud services, allowing them to tailor attacks to specific cloud environments and look for misconfigurations.

### **2. Resource Development**

Adversaries will set up the infrastructure and capabilities needed for their operation.

-   **T1586.002 - Compromise Accounts: Email Accounts:**
    -   **Relevance:** An attacker could compromise the personal email accounts of employees or third-party vendors (like the MSSP or Temenos support staff). These compromised accounts can then be used to send highly convincing phishing emails that appear to come from a trusted source, increasing the likelihood of success.
-   **T1583.006 - Acquire Infrastructure: Cloud Infrastructure:**
    -   **Relevance:** Given NordFinans' heavy use of Microsoft Azure, sophisticated adversaries will likely set up their command-and-control (C2) infrastructure within Azure. This helps their malicious traffic blend in with legitimate traffic to and from NordFinans' own Azure tenants, making detection by the SIEM and firewalls more challenging.
-   **T1587.001 - Develop Capabilities: Malware:**
    -   **Relevance:** Financially motivated groups may develop custom malware specifically designed to interact with or exploit vulnerabilities in the Temenos T24 core banking system or the SWIFT Alliance Lite2 environment. The "legacy components" and "older Java versions" mentioned in the profile represent a prime target for such custom exploit development.

### **3. Initial Access**

This is the critical phase where adversaries gain their first foothold in the network.

-   **T1566 - Phishing:**
    -   **Relevance:** With 2,000 employees using Microsoft 365, phishing is the most likely initial access vector. A successful phish could lead to the compromise of employee credentials, which, given the hybrid AD environment, could grant access to both cloud and on-premises resources.
-   **T1133 - External Remote Services:**
    -   **Relevance:** The Cisco AnyConnect VPN is a direct gateway into the internal network. A zero-day vulnerability in the VPN appliance or the compromise of a user's credentials (especially if that user's account is not yet protected by MFA for internal apps) could provide an attacker with significant access.
-   **T1190 - Exploit Public-Facing Application:**
    -   **Relevance:** The .NET-based web banking portal and the Open Banking APIs are complex applications exposed to the internet. A vulnerability like SQL injection, remote code execution, or an authentication bypass could grant an adversary direct access to backend systems in Azure.
-   **T1199 - Trusted Relationship:**
    -   **Relevance:** NordFinans relies on numerous third parties (Temenos, MSSP, Euronet). A compromise at one of these vendors could be used to pivot into the bank's network, for example, through a trusted VPN connection used for vendor support of the Temenos T24 system. The maturing third-party risk program indicates this is a significant area of exposure.

### **4. Execution**

Once inside, adversaries need to run their malicious code.

-   **T1059.001 - Command and Scripting Interpreter: PowerShell:**
    -   **Relevance:** PowerShell is installed on all Windows 11 endpoints and Windows Servers. It is a powerful tool for "living off the land," allowing attackers to perform discovery, lateral movement, and data collection while blending in with legitimate administrative activity, making it difficult for the SOC to detect.
-   **T1059.004 - Command and Scripting Interpreter: Unix Shell:**
    -   **Relevance:** The on-premises core banking system runs on RHEL. An attacker gaining access to these critical servers would use standard shell commands (`bash`, `sh`) to execute tools, manipulate files, and interact with the Oracle database.
-   **T1053.005 - Scheduled Task/Job: Scheduled Task:**
    -   **Relevance:** A common technique for executing malicious payloads and establishing persistence. An attacker could create a scheduled task on a compromised server or workstation to run their malware at a specific time or on system startup, ensuring their code runs even after a reboot.

### **5. Persistence**

Adversaries will establish methods to maintain their access over time.

-   **T1078 - Valid Accounts:**
    -   **Relevance:** This is a critical threat. By using stolen but legitimate credentials (e.g., from a successful phishing attack), an adversary can maintain access that is very difficult to detect. The known gap of "MFA not universally enforced for internal applications" makes this technique particularly dangerous, as a compromised account could be used to access multiple systems without further checks.
-   **T1136.001 - Create Account: Local Account:**
    -   **Relevance:** If an attacker gains administrative privileges on a server (e.g., a domain controller or a core banking server), they can create a new local or domain account. This "backdoor" account provides them with persistent access even if the initially compromised account is discovered and disabled.
-   **T1098.001 - Account Manipulation: Additional Cloud Credentials:**
    -   **Relevance:** In the Azure AD environment, an attacker with sufficient privileges could add a new credential (e.g., a password or certificate) to an existing high-privilege service principal or application. This would grant them persistent access to cloud resources without altering the primary user's credentials.

### **6. Privilege Escalation**

Adversaries will attempt to gain higher-level permissions.

-   **T1068 - Exploitation for Privilege Escalation:**
    -   **Relevance:** The self-assessed concern about "legacy components" and "older Java versions" within the Temenos T24 customizations is a significant risk. An unpatched vulnerability in these components could allow an attacker with standard user access to the server to escalate their privileges to root or system administrator.
-   **T1548.002 - Abuse Elevation Control Mechanism: Bypass User Account Control:**
    -   **Relevance:** On the 2,200 Windows 11 endpoints, attackers will use well-known techniques to bypass User Account Control (UAC). This would allow them to run processes with administrative rights without triggering a prompt for the user, enabling them to disable security tools or install rootkits.
-   **T1558.003 - Steal or Forge Kerberos Tickets: Kerberoasting:**
    -   **Relevance:** In the hybrid AD environment, an adversary can request service tickets for service accounts and attempt to crack them offline. A successful attack could reveal the cleartext password for a high-privilege service account, granting them elevated access across the on-premises network.

### **7. Defense Evasion**

Adversaries will try to avoid detection by security controls.

-   **T1562.001 - Impair Defenses: Disable or Modify Tools:**
    -   **Relevance:** A sophisticated attacker with administrative rights on an endpoint will attempt to disable or tamper with the CrowdStrike Falcon EDR agent. This would blind the SOC and MSSP to their subsequent activities on that machine.
-   **T1027 - Obfuscated Files or Information:**
    -   **Relevance:** To bypass detection by CrowdStrike and Microsoft Defender for Office 365, adversaries will use packers, crypters, and other obfuscation techniques on their malware payloads. This changes the file signature, making it harder for signature-based detection to identify.
-   **T1036 - Masquerading:**
    -   **Relevance:** Attackers will rename their malicious tools and processes to mimic legitimate system files (e.g., `svchost.exe`, `lsass.exe`). This tactic is designed to fool SOC analysts who are manually reviewing process lists on a compromised host, helping the attacker remain undetected.

### **8. Credential Access**

Adversaries will steal account names and passwords.

-   **T1003.001 - OS Credential Dumping: LSASS Memory:**
    -   **Relevance:** A primary objective for an attacker who has gained administrative access to a Domain Controller. By dumping credentials from the LSASS process memory, they can obtain password hashes for all active users on the domain, enabling widespread lateral movement.
-   **T1528 - Steal Application Access Token:**
    -   **Relevance:** In the M365 and Azure environments, adversaries can steal access tokens from a user's browser session. These tokens can then be replayed to gain access to the user's cloud applications (like SharePoint or Outlook) while bypassing MFA, as the token proves an already-authenticated session.
-   **T1555 - Credentials from Password Stores:**
    -   **Relevance:** Employees often store credentials in web browsers. An attacker on an endpoint can run tools to extract these saved passwords, potentially revealing credentials for other internal or third-party systems.

### **9. Discovery**

Once inside, adversaries will explore the network to understand the environment and identify key assets.

-   **T1087.002 - Account Discovery: Domain Account:**
    -   **Relevance:** Using simple commands like `net user /domain`, an attacker can enumerate all user accounts in the Active Directory. This helps them identify high-value targets like members of the "Domain Admins," "CyberArk Admins," or "SWIFT Operators" groups.
-   **T1046 - Network Service Discovery:**
    -   **Relevance:** After gaining an initial foothold, the adversary will scan the internal network segments to find critical systems. Their primary targets will be the IP addresses of the Temenos T24 servers, Oracle databases, Domain Controllers, and the servers within the highly-restricted SWIFT secure zone.
-   **T1580 - Cloud Infrastructure Discovery:**
    -   **Relevance:** If an attacker compromises an account with Azure access, they will use PowerShell or Azure CLI to enumerate subscriptions, resource groups, virtual machines, storage accounts, and Key Vaults. This helps them map the cloud environment and identify where sensitive data is stored.

### **10. Lateral Movement**

Adversaries will move through the environment to expand their access.

-   **T1021.001 - Remote Services: Remote Desktop Protocol:**
    -   **Relevance:** RDP is a standard administrative tool and is likely enabled between servers in the on-premises data centers. Using compromised credentials, attackers will use RDP to move from less critical servers to more sensitive systems like the core banking servers.
-   **T1550.002 - Use Alternate Authentication Material: Pass the Hash:**
    -   **Relevance:** After dumping credentials from LSASS, an attacker does not need the cleartext password. They can use the NTLM hash directly to authenticate to other Windows systems on the network, a technique that is highly effective in traditional Active Directory environments.
-   **T1021.006 - Remote Services: Windows Remote Management:**
    -   **Relevance:** WinRM is another common method for administration and can be used for lateral movement. It is often less monitored than RDP, making it an attractive channel for attackers to execute commands on remote systems.

### **11. Collection**

Adversaries will gather data of interest to their goals.

-   **T1213 - Data from Information Repositories:**
    -   **Relevance:** This is a primary objective. An attacker who gains access to the core banking environment will query the Temenos T24 Oracle database to collect customer PII, account balances, and transaction histories.
-   **T1530 - Data from Cloud Storage:**
    -   **Relevance:** Sensitive financial reports, customer data, and strategic plans may be stored in SharePoint Online. An attacker with compromised M365 credentials could access and download this data.
-   **T1114 - Email Collection:**
    -   **Relevance:** By accessing an employee's mailbox via compromised M365 credentials, an attacker can collect sensitive information, internal communications, and credentials shared over email. They can also set up forwarding rules to discreetly exfiltrate new emails.

### **12. Command and Control**

Adversaries will communicate with their compromised systems to control them.

-   **T1071.001 - Application Layer Protocol: Web Protocols (HTTP/S):**
    -   **Relevance:** C2 communication over HTTPS (port 443) is extremely common because it blends in with legitimate web traffic and is encrypted, making it difficult for the Palo Alto firewalls and the SOC to inspect and identify as malicious.
-   **T1567.002 - Exfiltration Over Web Service: Exfiltration to Cloud Storage:**
    -   **Relevance:** An attacker may use legitimate services like Microsoft OneDrive or a public file-sharing site as a C2 channel. This is particularly difficult to detect in an organization that heavily uses M365, as the traffic appears legitimate.
-   **T1090 - Proxy:**
    -   **Relevance:** To exfiltrate data from a highly segmented network like the core banking or SWIFT zone, an attacker may proxy their C2 traffic through a compromised machine in a less-restricted zone (like the office LAN), using it as a stepping stone to the internet.

### **13. Exfiltration**

Adversaries will steal the collected data.

-   **T1041 - Exfiltration Over C2 Channel:**
    -   **Relevance:** For smaller amounts of data, the adversary may exfiltrate it slowly over their existing C2 channel. This "low and slow" approach is less likely to trigger network traffic volume alerts.
-   **T1567.002 - Exfiltration Over Web Service: Exfiltration to Cloud Storage:**
    -   **Relevance:** For large-scale data theft (e.g., a dump of the customer database), an attacker is likely to compress and encrypt the data into an archive and upload it to an anonymous account on a public cloud storage platform (e.g., Azure Blob Storage, Dropbox).
-   **T1048 - Exfiltration Over Alternative Protocol:**
    -   **Relevance:** To bypass firewall rules focused on HTTP/S, an attacker might attempt to exfiltrate data over less common protocols like DNS or ICMP, which may not be as closely monitored.

### **14. Impact**

Adversaries will disrupt, destroy, or manipulate systems and data to achieve their goals.

-   **T1486 - Data Encrypted for Impact:**
    -   **Relevance:** Ransomware is a critical threat. An attacker could encrypt servers in both the on-premises data centers (targeting the core banking database) and in Azure. The financial and reputational impact of the core banking system being unavailable would be catastrophic.
-   **T1565.002 - Data Manipulation: Transmitted Data Manipulation:**
    -   **Relevance:** A sophisticated financial threat actor's ultimate goal may be theft. By gaining access to the SWIFT or core banking systems, they could manipulate, delete, or insert fraudulent payment messages and transactions. This directly targets the integrity of the bank's most critical function.
-   **T1499.004 - Endpoint Denial of Service: Application Exhaustion Flood:**
    -   **Relevance:** Adversaries could launch a denial-of-service attack against the web banking portal or the Open Banking APIs. This could overwhelm the Azure App Service or API Management gateway, making services unavailable to customers and TPPs, causing significant reputational damage and potential regulatory fines.
-   **T1491 - Defacement:**
    -   **Relevance:** To cause reputational harm, an attacker could deface the public-facing web banking portal. This type of attack erodes customer trust, a critical asset for any financial institution.
