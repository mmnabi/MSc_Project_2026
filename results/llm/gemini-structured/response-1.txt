### **Cybersecurity Threat Identification for NordFinans Bank AB**

This analysis identifies relevant cyber threats to NordFinans Bank AB by mapping its specific operational context, infrastructure, and security posture to the MITRE ATT&CK Enterprise framework.

---

### **1. Reconnaissance (TA0043)**

Adversaries will gather information to plan an attack. As a publicly-traded financial institution, NordFinans is a high-value target with a significant public footprint.

*   **T1595: Active Scanning**
    *   **Relevance:** Adversaries will actively scan NordFinans' external perimeter, which includes the Azure Front Door protecting the web banking portal, the Cisco AnyConnect VPN endpoints, and the Azure API Management gateway for Open Banking. These scans will look for vulnerabilities, open ports, and misconfigurations that could be exploited for initial access.
*   **T1593: Search Open Websites/Domains**
    *   **Relevance:** Attackers will scrape public sources like the NordFinans corporate website, Nasdaq Stockholm listings, and social media (especially LinkedIn) to gather employee names, roles, and email address formats. This information is critical for building targeted spearphishing campaigns against the bank's ~2,000 employees.
*   **T1592: Gather Victim Host Information**
    *   **Relevance:** Adversaries will search for technical details in job postings (e.g., "Temenos T24 Specialist," "Azure DevOps Engineer," "Palo Alto Administrator") and technical blogs to build a precise map of the bank's internal technology stack. This allows them to pre-select exploits and tools tailored to NordFinans' environment.
*   **T1589: Gather Victim Identity Information**
    *   **Relevance:** Attackers will search for breached credentials from other services that might be reused by NordFinans employees. This information can be used in credential stuffing attacks against the VPN or M365 services.

---

### **2. Resource Development (TA0042)**

Adversaries will establish resources to support their operations.

*   **T1584: Compromise Infrastructure**
    *   **Relevance:** Adversaries may compromise the infrastructure of one of NordFinans' less secure third-party vendors (as the bank's third-party risk program is "maturing"). This could provide a trusted path into the bank's network, for example, by compromising a vendor with VPN access for support (like Temenos).
*   **T1587: Develop Capabilities**
    *   **Relevance:** Financially motivated groups will develop or purchase custom malware designed to target financial systems. This could include malware specifically crafted to interact with the Temenos T24 core banking system or the SWIFT Alliance Lite2 environment to automate fraudulent transactions, while evading detection by CrowdStrike Falcon.
*   **T1583: Acquire Infrastructure**
    *   **Relevance:** Attackers will set up command and control (C2) servers and phishing domains that mimic legitimate NordFinans or Microsoft 365 services (e.g., `nordfinans-login.com`, `m365-secure.se`). This infrastructure is essential for hosting phishing sites and managing compromised systems.

---

### **3. Initial Access (TA0001)**

Adversaries will attempt to get into the network. This is a critical stage where multiple entry points exist.

*   **T1566: Phishing**
    *   **Relevance:** This is the most likely initial access vector. Spearphishing campaigns will target employees with emails containing malicious links or attachments. Given the use of Microsoft 365, these campaigns will likely impersonate M365 notifications to harvest credentials or trick users into running malware on their Windows 11 endpoints.
*   **T1190: Exploit Public-Facing Application**
    *   **Relevance:** The .NET-based web banking portal, the mobile banking backend APIs, and the Open Banking APIs are all potential targets. A vulnerability like a SQL injection, remote code execution (RCE), or an authentication bypass in these Azure-hosted applications could give an attacker direct access to sensitive systems or customer data.
*   **T1133: External Remote Services**
    *   **Relevance:** The Cisco AnyConnect VPN is a direct entry point to the internal network. Adversaries will target it with brute-force attacks, credential stuffing, or by exploiting zero-day vulnerabilities. While MFA is a strong control, it can be bypassed via MFA fatigue attacks or social engineering.
*   **T1195: Supply Chain Compromise**
    *   **Relevance:** An attacker could compromise a software or service provider used by NordFinans. Key targets include the Azure DevOps CI/CD pipeline (to inject malicious code into applications) or the MSSP that provides after-hours SOC monitoring (to gain access to security tools and logs).

---

### **4. Execution (TA0002)**

Adversaries will try to run malicious code on a compromised system.

*   **T1059.001: PowerShell**
    *   **Relevance:** PowerShell is native to all Windows 11 endpoints and Windows Servers. Adversaries will use it for "living off the land" techniques—executing commands, downloading additional payloads, and performing reconnaissance without dropping a binary file to disk, making it harder for EDR solutions like CrowdStrike to detect.
*   **T1059.003: Windows Command Shell**
    *   **Relevance:** The command prompt (`cmd.exe`) remains a common tool for attackers to execute scripts and built-in system commands for discovery and lateral movement.
*   **T1204: User Execution**
    *   **Relevance:** This technique is tied directly to phishing. An attacker relies on a user clicking a malicious link or opening a weaponized document, which then triggers the execution of the initial payload on their corporate workstation.

---

### **5. Persistence (TA0003)**

Adversaries will try to maintain their foothold in the network.

*   **T1078: Valid Accounts**
    *   **Relevance:** Once an adversary has stolen credentials, using them to log in is the most effective form of persistence as it blends in with legitimate activity. The hybrid AD environment with ~3,500 accounts provides a large pool of potential targets.
*   **T1136.003: Create Account: Cloud Account**
    *   **Relevance:** An attacker with sufficient privileges in Azure AD could create a new guest or member account. This rogue account, potentially with elevated roles, could be used for long-term, stealthy access to the bank's cloud resources.
*   **T1053.005: Scheduled Task/Job: Scheduled Task**
    *   **Relevance:** On a compromised Windows endpoint or server, an adversary can create a scheduled task to re-run their malware periodically or upon system startup, ensuring their access survives reboots.

---

### **6. Privilege Escalation (TA0004)**

Adversaries will try to gain higher-level permissions.

*   **T1558.003: Kerberoasting**
    *   **Relevance:** Within the on-premises Active Directory environment, an attacker with any domain user account can request service tickets for service accounts. They can then attempt to crack these tickets offline to reveal the service account's plaintext password. This is a highly effective way to escalate privileges.
*   **T1068: Exploitation for Privilege Escalation**
    *   **Relevance:** An attacker could exploit a known but unpatched vulnerability on a Windows Server or RHEL server (e.g., in the on-premises core banking environment) to escalate from a low-privilege user to SYSTEM or root. The "legacy components" in the T24 environment are a particular concern here.
*   **T1078: Valid Accounts**
    *   **Relevance:** This is particularly relevant due to the known gap: "MFA not universally enforced for internal applications." An attacker who has compromised a standard user account can move laterally to an internal system and log in as a privileged user (whose credentials they have stolen) without being challenged by MFA, thereby escalating their privileges.

---

### **7. Defense Evasion (TA0005)**

Adversaries will try to avoid being detected.

*   **T1562.001: Impair Defenses: Disable or Modify Tools**
    *   **Relevance:** Adversaries will attempt to disable or tamper with CrowdStrike Falcon on endpoints to prevent their actions from being detected or blocked. They may also try to alter firewall rules on the Palo Alto devices if they gain administrative access.
*   **T1027: Obfuscated Files or Information**
    *   **Relevance:** To bypass security controls, attackers will use packers, crypters, or other obfuscation methods on their malware payloads to change their signatures and evade detection by EDR and anti-virus solutions.
*   **T1550.002: Use Alternate Authentication Material: Pass the Hash**
    *   **Relevance:** In the Windows-heavy on-prem environment, attackers can use Pass-the-Hash techniques to authenticate to other systems without needing the plaintext password. This bypasses traditional credential-based detection and is very effective for lateral movement.
*   **T1070.004: Indicator Removal: File Deletion**
    *   **Relevance:** After executing payloads or scripts, attackers will delete them from disk to clean up their tracks and make forensic analysis more difficult for the internal SOC or MSSP.

---

### **8. Credential Access (TA0006)**

Adversaries will try to steal account names and passwords.

*   **T1003.001: OS Credential Dumping: LSASS Memory**
    *   **Relevance:** A primary objective after gaining access to a Windows system is to dump credentials from the LSASS process memory using tools like Mimikatz. This can yield plaintext passwords, hashes, and Kerberos tickets for users logged into the system, including privileged administrators.
*   **T1110.003: Brute Force: Password Spraying**
    *   **Relevance:** Adversaries will perform slow password spraying attacks against internal applications and services that lack MFA. Using common passwords (e.g., `Summer2024!`) against a large list of user accounts can often yield a valid credential without triggering account lockout policies.
*   **T1528: Steal Application Access Token**
    *   **Relevance:** In the hybrid M365/Azure environment, attackers can steal access tokens (e.g., from a user's browser session) to gain access to cloud resources like SharePoint Online, Exchange Online, and Azure portals without needing a password or MFA.

---

### **9. Discovery (TA0007)**

Adversaries will try to figure out your environment.

*   **T1087.002: Account Discovery: Domain Account**
    *   **Relevance:** Once inside the network, an attacker will run commands like `net user /domain` to enumerate all accounts in the Active Directory. This helps them identify high-value targets like domain admins, SWIFT operators, and database administrators managed in CyberArk.
*   **T1046: Network Service Discovery**
    *   **Relevance:** Attackers will scan the internal network segments to identify live hosts and open services. They will be specifically looking for the Oracle database servers for the Temenos core banking system, the SWIFT secure zone, and domain controllers.
*   **T1526: Cloud Service Discovery**
    *   **Relevance:** If the attacker gains access to an Azure-credentialed account, they will use Azure CLI or PowerShell to enumerate subscriptions, resource groups, VMs, storage accounts, and key vaults to understand the cloud environment and identify high-value assets.

---

### **10. Lateral Movement (TA0008)**

Adversaries will try to move through your environment.

*   **T1021.001: Remote Services: Remote Desktop Protocol**
    *   **Relevance:** RDP is a common protocol used by administrators and will be abused by attackers to move between Windows servers. An attacker with valid credentials can use RDP to pivot from a less critical server into more sensitive zones, such as the management zone or the periphery of the core banking network.
*   **T1570: Lateral Tool Transfer**
    *   **Relevance:** Adversaries will use legitimate network protocols like SMB to copy their tools (e.g., Mimikatz, scanners) from their initial point of compromise to other systems in the network, staging them for further attacks.
*   **T1210: Exploitation of Remote Services**
    *   **Relevance:** Attackers may exploit vulnerabilities in services running on internal servers, such as SMB (e.g., EternalBlue), to move laterally without needing credentials.

---

### **11. Collection (TA0009)**

Adversaries will gather data of interest to their goal.

*   **T1213.002: Data from Information Repositories: Databases**
    *   **Relevance:** This is a primary objective. An attacker who has reached the core banking zone will attempt to connect to the Temenos Oracle database to exfiltrate customer financial data, transaction histories, and PII. This directly impacts Key Asset #1.
*   **T1114.002: Email Collection: Remote Public or Shared Mailbox**
    *   **Relevance:** Attackers will access Exchange Online to search through emails for sensitive information, such as internal procedures for wire transfers, credentials, or confidential M&A documents.
*   **T1530: Data from Cloud Storage Object**
    *   **Relevance:** Data stored in Azure Blob Storage or SharePoint Online is a prime target. Adversaries will look for improperly secured storage containers or access SharePoint sites containing sensitive financial reports or customer GDPR-related data.

---

### **12. Command and Control (TA0011)**

Adversaries will try to communicate with compromised systems to control them.

*   **T1071.001: Application Layer Protocol: Web Protocols**
    *   **Relevance:** Adversaries will use HTTP/HTTPS for their C2 traffic because it is allowed through almost all firewalls (including the Palo Alto NGFWs) and blends in with legitimate web traffic, making it difficult for the SOC/MSSP to detect.
*   **T1573.001: Encrypted Channel: Symmetric Cryptography**
    *   **Relevance:** C2 communications will be encrypted to prevent network inspection tools from analyzing the content of the traffic. This makes it challenging to understand what commands are being sent or what data is being exfiltrated.
*   **T1105: Ingress Tool Transfer**
    *   **Relevance:** The C2 channel will be used to download additional malicious tools onto compromised systems as the attack progresses, allowing the adversary to bring in specialized tools only when needed.

---

### **13. Exfiltration (TA0010)**

Adversaries will try to steal data.

*   **T1041: Exfiltration Over C2 Channel**
    *   **Relevance:** For smaller amounts of data, attackers will typically exfiltrate it slowly over their existing C2 channel. This is often done in small, encrypted chunks to avoid triggering alerts for large data transfers.
*   **T1567.002: Exfiltration Over Web Service: Exfiltration to Cloud Storage**
    *   **Relevance:** To exfiltrate large volumes of data (e.g., a database dump from the core banking system), attackers may upload it to a legitimate third-party cloud storage service like Dropbox, Google Drive, or a rogue Azure storage account. This traffic can be difficult to distinguish from legitimate use.
*   **T1048: Exfiltration Over Alternative Protocol**
    *   **Relevance:** Attackers may use protocols like DNS tunneling to exfiltrate data, as DNS requests are often less scrutinized than HTTP/HTTPS traffic.

---

### **14. Impact (TA0040)**

Adversaries will try to manipulate, interrupt, or destroy your systems and data.

*   **T1486: Data Encrypted for Impact**
    *   **Relevance:** Ransomware is a critical threat. An attacker could encrypt key on-premises servers (Windows and RHEL), Azure VMs, and potentially the core banking database, causing a complete operational shutdown. The bank's IR plan and air-gapped backups are the primary defenses here.
*   **T1487: Financial Theft**
    *   **Relevance:** For a bank, this is the ultimate goal for many attackers. After gaining access to the SWIFT environment or the Temenos T24 system, adversaries will attempt to initiate fraudulent wire transfers to accounts they control. This represents a direct and immediate financial loss.
*   **T1489: Service Stop**
    *   **Relevance:** A disruptive attacker could shut down the core banking system, the web/mobile banking platforms, or the ATM processing connections. This would prevent customers from accessing their accounts, causing significant reputational damage and regulatory scrutiny under DORA.
*   **T1485: Data Destruction**
    *   **Relevance:** A more malicious or state-sponsored actor might choose to destroy data rather than ransom it. This could involve wiping servers, corrupting the Oracle database, or deleting cloud storage and backups, with the goal of causing maximum, potentially unrecoverable, damage to the bank.
