# Synthetic Organizational Profile: NordFinans Bank AB

---

## Organization Overview

**Name:** NordFinans Bank AB (fictional)
**Type:** Mid-size retail and commercial bank
**Headquarters:** Stockholm, Sweden
**Employees:** ~2,000 (1,600 in Sweden, 400 across Nordic branch offices)
**Annual Revenue:** ~€800M
**Customer Base:** ~500,000 retail customers, ~12,000 SME business customers

NordFinans Bank AB is a mid-size Nordic bank providing retail banking, SME commercial lending, payment services, and wealth management. The bank operates primarily in Sweden with branch offices in Norway, Denmark, and Finland. It is publicly traded on Nasdaq Stockholm (Mid Cap).

---

## Regulatory Environment

- **PSD2 (Payment Services Directive 2):** NordFinans operates Open Banking APIs allowing authorized third-party providers (TPPs) access to customer account data and payment initiation. API gateway managed internally.
- **GDPR (General Data Protection Regulation):** Processes personal data for ~500,000 customers across four jurisdictions. A Data Protection Officer (DPO) reports to the Chief Compliance Officer.
- **Swedish Financial Supervisory Authority (Finansinspektionen):** Subject to FFFS 2014:5 (IT governance) and EBA Guidelines on ICT and security risk management.
- **DORA (Digital Operational Resilience Act):** Compliance deadline January 2025. Currently implementing ICT risk management framework, third-party risk oversight, and incident reporting capabilities.
- **SWIFT Customer Security Programme (CSP):** Mandatory self-attestation against SWIFT CSCF v2024. Annual independent assessment performed.

---

## IT Infrastructure

### Network Architecture

- **Hybrid cloud model:** Core banking system (Temenos T24) runs on-premises in two co-located data centers (Stockholm, Gothenburg) with active-passive failover. Supporting services progressively migrating to Microsoft Azure (North Europe region).
- **Internal network:** Segmented into zones — core banking, office LAN, DMZ, development, and management. Inter-zone traffic filtered via Palo Alto next-generation firewalls.
- **Remote access:** Employees use Cisco AnyConnect VPN with MFA (Microsoft Authenticator) for remote work. Approximately 40% of staff work hybrid (2–3 days remote).
- **Branch connectivity:** Branch offices connected via MPLS WAN with IPsec backup tunnels.

### Customer-Facing Services

- **Web banking portal:** .NET-based application hosted on Azure App Service behind Azure Front Door (WAF + CDN). Handles ~200,000 active monthly users.
- **Mobile banking app:** Native iOS and Android apps communicating via RESTful APIs to Azure-hosted backend microservices. Biometric authentication supported.
- **Open Banking APIs:** PSD2-compliant APIs (account information, payment initiation) exposed via Azure API Management with OAuth 2.0 / OpenID Connect.
- **ATM network:** ~120 ATMs across Sweden, managed by a third-party processor (Euronet). Connection via dedicated VPN tunnels to core banking.

### Internal Systems

- **Core banking:** Temenos T24 on-premises (Linux/Oracle DB). Batch processing overnight; real-time transaction processing during business hours.
- **SWIFT connectivity:** Alliance Lite2 for interbank messaging. Dedicated SWIFT secure zone with hardware security modules (HSMs) for key management.
- **Email and collaboration:** Microsoft 365 (Exchange Online, Teams, SharePoint Online). Email protected by Microsoft Defender for Office 365.
- **Active Directory:** Hybrid AD with Azure AD Connect. ~3,500 accounts (employees, contractors, service accounts).
- **ERP/HR:** SAP S/4HANA Cloud for financials and HR.
- **Development:** Azure DevOps for CI/CD pipelines. Internal applications developed in .NET, Java, and Python.

### Endpoint Environment

- **Workstations:** ~2,200 Windows 11 endpoints managed via Microsoft Intune. CrowdStrike Falcon for EDR.
- **Servers:** ~150 on-premises servers (Windows Server 2019/2022, RHEL 8/9). ~80 Azure VMs.
- **Mobile devices:** ~800 corporate-managed iOS/Android devices via Intune MDM.

---

## Security Posture

### Security Operations

- **SOC:** Small internal SOC team of 5 analysts (1 lead, 4 analysts) operating during business hours (08:00–18:00 CET). After-hours monitoring outsourced to a Nordic MSSP (Managed Security Service Provider).
- **SIEM:** Microsoft Sentinel (cloud-native) ingesting logs from Azure, M365, on-premises firewalls, core banking (via syslog), and CrowdStrike.
- **Vulnerability management:** Qualys for quarterly external scans and monthly internal scans. Patch management via WSUS (on-premises) and Intune (cloud).
- **Penetration testing:** Annual external penetration test by a contracted third party. Scope covers web banking, mobile app, and external perimeter.

### Identity and Access Management

- **Privileged access:** CyberArk for privileged account vaulting covering ~200 privileged accounts (domain admins, database admins, SWIFT operators).
- **MFA:** Enforced for all remote access, all cloud admin portals, and SWIFT operations. Not yet enforced for all internal application access.
- **Access reviews:** Quarterly access recertification for critical systems (core banking, SWIFT, AD admin groups).

### Incident Response

- **IR plan:** Documented and tested annually via tabletop exercise. Last exercise focused on ransomware scenario (Q3 2025).
- **CERT relationship:** Member of FinansCERT (Nordic financial sector CERT). Participates in sector-wide threat intelligence sharing.
- **Backup and recovery:** Daily incremental backups, weekly full backups. Core banking RPO: 1 hour, RTO: 4 hours. Backup infrastructure partially air-gapped (tape storage offsite).

## Key Assets (Priority Order)

1. **Core banking system and customer financial data** — transactional integrity and data confidentiality.
2. **SWIFT messaging infrastructure** — direct financial fraud risk.
3. **Customer PII across all systems** — regulatory (GDPR) and reputational exposure.
4. **Web/mobile banking platforms** — service availability and customer trust.
5. **Active Directory and identity infrastructure** — foundation for access to all other assets.
6. **Open Banking APIs** — regulatory compliance (PSD2) and third-party trust.
7. **Development pipelines (Azure DevOps)** — supply chain integrity for internally developed applications.

## Known Gaps and Concerns (Self-Assessed)

- MFA not universally enforced for internal applications.
- SOC capacity limited — business-hours-only internal coverage creates detection lag for after-hours incidents.
- Legacy components in core banking (some T24 customizations running on older Java versions).
- Third-party risk management program maturing — vendor assessments not yet comprehensive across all suppliers.
- Cloud migration in progress — security controls for Azure workloads still being standardized.
- Limited OT/IoT footprint (building management systems, some branch physical security cameras) — not actively monitored by SOC.
