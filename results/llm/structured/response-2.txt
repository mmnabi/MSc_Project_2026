# MITRE ATT&CK Enterprise Threat Analysis
## NordFinans Bank AB

---

## Tactic 1 — Reconnaissance

Reconnaissance is largely passive and difficult to detect. For a publicly traded bank with regulatory disclosure requirements, PSD2 API documentation, and a visible technology footprint, the reconnaissance surface is broad.

---

**T1591 — Gather Victim Org Information**
*(Sub-techniques: T1591.002 Business Relationships, T1591.004 Identify Roles)*

NordFinans's status as a Nasdaq Stockholm Mid Cap company means annual reports, sustainability disclosures, and regulatory filings are publicly available. These documents name technology vendors (Temenos, Microsoft, SWIFT), describe the hybrid cloud strategy, and identify senior personnel (CISO, DPO, CFO). Adversaries — particularly nation-state and FIN-group actors — routinely exploit these disclosures to map organizational relationships and prioritize targets within the supply chain. The named third-party vendors (Euronet, Nordic MSSP, Temenos) become pivot points for supply chain reconnaissance.

---

**T1592 — Gather Victim Host Information**
*(Sub-techniques: T1592.002 Software, T1592.003 Firmware)*

Job postings are a primary source. A bank posting roles for "CrowdStrike Falcon administrator," "Azure Sentinel SIEM engineer," "CyberArk PAM engineer," or "Temenos T24 developer" unambiguously reveals the security tool stack to any adversary willing to spend 30 minutes on LinkedIn. This allows attackers to pre-select evasion techniques effective against CrowdStrike Falcon, craft payloads that avoid Sentinel detection rules, and identify gaps (e.g., the absence of universal MFA on internal applications is implied by job descriptions referencing "MFA rollout" initiatives). The .NET/Java/Python development stack revealed through Azure DevOps job postings also informs payload selection.

---

**T1593 — Search Open Websites/Domains**
*(Sub-techniques: T1593.001 Social Media, T1593.002 Search Engines, T1593.003 Code Repositories)*

The PSD2 Open Banking API is, by regulatory design, publicly documented. NordFinans must publish its API endpoints, supported OAuth 2.0 flows, and onboarding processes for Third Party Providers (TPPs). Adversaries can register as a TPP, study the API specification, and conduct legitimate-looking reconnaissance of the API gateway before any attack begins. Additionally, if any internal code has been inadvertently pushed to public repositories (common during cloud migration projects using Azure DevOps), secrets, configuration files, or API keys may be exposed. Certificate Transparency logs for `nordfinans.se` subdomains reveal the full scope of external infrastructure.

---

**T1596 — Search Open Technical Databases**
*(Sub-techniques: T1596.001 DNS/Passive DNS, T1596.002 WHOIS, T1596.005 Scan Databases)*

Shodan and Censys scans of NordFinans's IP ranges would enumerate Azure Front Door WAF endpoints, VPN concentrators, mail exchange records, and any misconfigured Azure services with public exposure. Passive DNS analysis reveals historical IP-to-hostname mappings that may expose legacy infrastructure not covered by the annual external penetration test scope. SWIFT-related infrastructure, while within a dedicated secure zone, may have indirect exposure through management interfaces.

---

**T1598 — Phishing for Information**
*(Sub-technique: T1598.003 Spearphishing Link)*

Pre-attack credential harvesting against NordFinans employees via convincing lookalike pages (mimicking Microsoft 365 login or Azure portal) is a precursor to many initial access techniques. Given that approximately 40% of staff work hybrid with Cisco AnyConnect VPN, adversaries can craft highly targeted pretexts around VPN credential resets or M365 MFA re-enrollment, lowering staff suspicion.

---

## Tactic 2 — Resource Development

Resource development represents the adversary's preparation phase. Sophisticated actors — particularly those targeting SWIFT infrastructure or conducting ransomware campaigns — invest significantly here.

---

**T1583 — Acquire Infrastructure**
*(Sub-techniques: T1583.001 Domains, T1583.003 Virtual Private Server, T1583.006 Web Services)*

Adversaries targeting NordFinans will register typosquat domains (e.g., `nord-finans.se`, `nordfinansbank.com`) for use in phishing campaigns and C2 infrastructure. For SWIFT-targeting actors (consistent with FIN groups and nation-state actors who have historically targeted SWIFT — see Bangladesh Bank, Banco de Chile), dedicated C2 infrastructure hosted on VPS providers in jurisdictions with poor cooperation records is standard tradecraft. Azure-hosted C2 is increasingly common to blend into NordFinans's own outbound Azure traffic.

---

**T1585 — Establish Accounts**
*(Sub-techniques: T1585.001 Social Media Accounts, T1585.002 Email Accounts)*

For Open Banking API abuse, adversaries may legitimately register as a TPP with the Swedish Financial Supervisory Authority or with NordFinans directly, obtaining valid OAuth 2.0 credentials that provide authorized access to customer account information APIs. This is not a hypothetical — regulatory open banking frameworks create a legitimate onboarding pathway that adversaries can exploit without triggering technical alarms, since their traffic would appear as authorized TPP activity.

---

**T1587 — Develop Capabilities**
*(Sub-techniques: T1587.001 Malware, T1587.003 Digital Certificates)*

Groups targeting Temenos T24 banking platforms have developed custom implants tailored to the platform's architecture (Linux/Oracle DB). Given that T24 is widely deployed across Nordic and European banks, a modular implant targeting T24 batch processing interfaces or Oracle DB connections would be highly reusable. Self-signed certificates mimicking NordFinans's internal PKI can be used to authenticate malicious payloads or man-in-the-middle internal HTTPS traffic during cloud migration phases when certificate validation controls may be inconsistently applied.

---

**T1588 — Obtain Capabilities**
*(Sub-techniques: T1588.001 Malware, T1588.002 Tool, T1588.005 Exploits)*

The commercial exploit and crimeware ecosystem is well-stocked for NordFinans's technology stack. Banking trojans (IcedID, QakBot variants) with M365 credential-harvesting modules are commercially available. CobaltStrike and Brute Ratel C4 licenses are sold to criminal actors. Exploits for older Java versions — relevant given the legacy Java components in T24 customizations — can be obtained via exploit brokers. The combination of a known core banking platform (T24) and a known Java vulnerability surface makes capability acquisition straightforward for well-resourced adversaries.

---

## Tactic 3 — Initial Access

Given NordFinans's attack surface — external web services, Open Banking APIs, M365, hybrid VPN access, and multiple third-party vendor connections — the initial access tactic has numerous viable pathways.

---

**T1566 — Phishing**
*(Sub-techniques: T1566.001 Spearphishing Attachment, T1566.002 Spearphishing Link, T1566.003 Spearphishing via Service)*

Spearphishing is the highest-probability initial access vector. NordFinans's M365 environment (Exchange Online with Microsoft Defender for Office 365) is the primary target surface. Microsoft Defender for Office 365 is effective but not impenetrable — HTML smuggling, QR-code phishing (bypassing URL scanning), and adversary-in-the-middle (AiTM) phishing kits that relay authentication tokens in real time are proven bypasses. Spearphishing via Microsoft Teams (T1566.003) is specifically relevant because Teams federation may allow external users to message NordFinans employees, and Teams messages receive less scrutiny than email. Finance and treasury staff handling SWIFT transactions are high-value spearphishing targets due to their privileged access.

---

**T1190 — Exploit Public-Facing Application**

The web banking portal (.NET on Azure App Service behind Azure Front Door WAF) and the Open Banking API gateway (Azure API Management with OAuth 2.0) represent the most directly exploitable external attack surface. Specific risks include: injection vulnerabilities in the .NET application; OAuth 2.0 implementation flaws in the PSD2 API (e.g., authorization code interception, token leakage); and API Management misconfigurations that expose internal microservice endpoints. The WAF mitigates known signatures but is not a guarantee against zero-day or novel attack patterns. Annual penetration testing scope covers this surface, but quarterly Qualys scans do not substitute for continuous dynamic testing of application logic.

---

**T1133 — External Remote Services**

Cisco AnyConnect VPN with MFA (Microsoft Authenticator) is the remote access pathway for approximately 800+ hybrid employees. This is a well-hardened control, but MFA can be bypassed through AiTM phishing (capturing session cookies post-authentication), MFA fatigue/push bombing attacks, and SIM-swapping of phone-based authenticators. If an attacker obtains valid VPN credentials (e.g., via phishing) and can defeat MFA through one of these mechanisms, they gain network-level access equivalent to being on the internal LAN — bypassing perimeter controls and accessing the internal AD environment.

---

**T1195 — Supply Chain Compromise**
*(Sub-techniques: T1195.001 Compromise Software Dependencies and Development Tools, T1195.002 Compromise Software Supply Chain)*

NordFinans's Azure DevOps CI/CD pipeline is explicitly identified as a key asset. Supply chain compromise via malicious packages in the .NET/Java/Python dependency chain (e.g., typosquatted NuGet or PyPI packages) could introduce backdoors into internally developed applications before deployment. This is particularly dangerous because the compromised code would carry NordFinans's own signing certificates and be deployed through legitimate change management processes, bypassing most security controls.

---

**T1199 — Trusted Relationship**

Four third-party vendors maintain persistent, privileged connectivity to NordFinans's network: Temenos (support VPN to T24), Euronet (dedicated VPN to ATM network), the Nordic MSSP (VPN + API integration), and CyberArk (cloud component). Each of these trusted connections is a potential lateral entry point. The Temenos vendor support VPN is particularly high-risk: support VPN connections often have broad access to production systems, may not be subject to the same MFA enforcement as employee VPNs, and the vendor's own security posture may be weaker than the bank's. Nation-state actors have repeatedly exploited vendor VPN access (see SolarWinds, Kaseya) to compromise downstream customers. The Nordic MSSP is similarly high-value — compromise of the MSSP gives after-hours visibility into NordFinans's Sentinel alerts and CrowdStrike telemetry, allowing attackers to operate with awareness of detection thresholds.

---

**T1078 — Valid Accounts**
*(Sub-techniques: T1078.001 Default Accounts, T1078.002 Domain Accounts, T1078.004 Cloud Accounts)*

Credential stuffing attacks against the web banking portal and mobile app using credentials obtained from unrelated breaches (credential lists are commercially available) represent a persistent, low-sophistication but high-volume threat to 500,000 retail customers. For internal access, service accounts — of which there are an unknown number within the ~3,500 AD accounts — often have long-lived passwords, broad permissions, and no MFA enforcement, making them a high-value target once an attacker has a foothold.

---

## Tactic 4 — Execution

Once inside the environment, execution techniques allow the adversary to run their code. NordFinans's hybrid environment (Windows 11 endpoints, Windows/RHEL servers, Azure VMs) and cloud services create multiple execution surfaces.

---

**T1059 — Command and Scripting Interpreter**
*(Sub-techniques: T1059.001 PowerShell, T1059.003 Windows Command Shell, T1059.006 Python, T1059.007 JavaScript)*

PowerShell is the dominant execution vehicle on Windows 11 endpoints and Windows Server systems. While CrowdStrike Falcon provides behavioral detection for malicious PowerShell, living-off-the-land (LotL) techniques using built-in Windows binaries continue to evolve. On the Linux T24 servers (RHEL 8/9), Bash scripting is the execution vector. Python is relevant given that NordFinans developers write Python applications — a compromised developer workstation with Python installed is a ready execution platform. JavaScript/TypeScript execution within Azure Functions or Node.js microservices is relevant for cloud-hosted application compromise.

---

**T1204 — User Execution**
*(Sub-techniques: T1204.001 Malicious Link, T1204.002 Malicious File, T1204.003 Malicious Image)*

Microsoft Office macro execution is now partially mitigated by Microsoft's default macro blocking policy. However, ISO/IMG file delivery (bypassing mark-of-the-web), OneNote attachments embedding malicious scripts, and HTML smuggling delivering ZIP files remain active tradecraft. Given that approximately 40% of staff work hybrid and may use personal printers or USB devices at home, the attack surface extends beyond corporate-managed endpoints. Azure Virtual Machine images — relevant during the ongoing cloud migration — could be compromised at the image level (T1204.003), deploying malicious workloads through the normal VM provisioning process.

---

**T1072 — Software Deployment Tools**

Microsoft Intune manages ~2,200 Windows 11 endpoints and ~800 mobile devices. Azure DevOps runs CI/CD pipelines. If an adversary compromises an Intune administrator account or Azure DevOps pipeline service account, they can deploy malicious software to all managed endpoints simultaneously — a highly efficient execution mechanism that would appear as a legitimate software deployment to security monitoring. This is directly analogous to the SolarWinds attack vector applied to NordFinans's internal toolchain.

---

**T1053 — Scheduled Task/Job**
*(Sub-techniques: T1053.002 At, T1053.005 Scheduled Task, T1053.007 Container Orchestration Job)*

The Temenos T24 core banking system relies on batch processing overnight. Malicious scheduled tasks inserted into the batch processing queue would execute outside business hours — precisely when the internal SOC has handed off to the Nordic MSSP (a lower-capability monitoring tier). An attacker aware of the SOC's hours-of-operation model (discoverable through job postings or the MSSP's own marketing materials) would specifically time malicious execution for overnight hours.

---

**T1047 — Windows Management Instrumentation**

WMI is extensively used in Windows Server environments for remote administration and monitoring. It provides a built-in execution mechanism that is difficult to distinguish from legitimate administrative activity. In NordFinans's environment, where hybrid AD connects on-premises Windows Servers to Azure AD, WMI lateral execution is a high-probability technique for spreading from a compromised endpoint to servers in the core banking network zone.

---

## Tactic 5 — Persistence

Persistence is critical for attackers conducting slow-burn operations (e.g., SWIFT fraud reconnaissance, intellectual property theft, or pre-positioning for destructive attacks). NordFinans's hybrid environment creates multiple persistence surfaces.

---

**T1078 — Valid Accounts (Persistence Context)**
*(Sub-techniques: T1078.002 Domain Accounts, T1078.004 Cloud Accounts)*

Maintaining persistence through legitimate but compromised accounts is the most operationally silent approach. An attacker with a compromised domain account — particularly a service account not subject to routine access reviews — can maintain access indefinitely. Service accounts in the ~3,500 AD account population may have passwords that are years old and are not covered by quarterly access recertification (which focuses on critical systems). Azure AD service principals used by CI/CD pipelines or Azure workloads are similarly long-lived and often under-monitored.

---

**T1505 — Server Software Component**
*(Sub-technique: T1505.003 Web Shell)*

A web shell deployed to the Azure App Service hosting the web banking portal, or to an on-premises web server in the DMZ, provides persistent HTTP/HTTPS-accessible command execution that blends into normal web traffic. WAF rules detect known web shell signatures, but custom-obfuscated shells evade signature detection. Given that the annual penetration test covers the web banking surface, this vector is checked annually but not continuously.

---

**T1098 — Account Manipulation**
*(Sub-techniques: T1098.001 Additional Cloud Credentials, T1098.003 Additional Cloud Roles, T1098.005 Device Registration)*

In the Azure AD / hybrid AD environment, an attacker with sufficient privileges can add additional credentials to existing service principals (T1098.001), escalate the role assignments of a compromised account (T1098.003), or register a new device to bypass Conditional Access policies that rely on device compliance (T1098.005). These manipulations are subtle and may not trigger immediate alerts if Azure AD audit logging is not tuned to detect them — a realistic risk given that Sentinel cloud security controls are "still being standardized."

---

**T1543 — Create or Modify System Process**
*(Sub-technique: T1543.003 Windows Service)*

Creating a malicious Windows service on domain-joined servers provides persistence that survives reboots and is difficult to detect without baselining of installed services. On RHEL 8/9 servers hosting T24 components, malicious systemd service units achieve the same effect. CrowdStrike Falcon detects many service-based persistence mechanisms on Windows, but coverage of Linux servers depends on whether the Falcon Linux sensor is deployed — the profile does not confirm this.

---

**T1546 — Event Triggered Execution**
*(Sub-technique: T1546.003 Windows Management Instrumentation Event Subscription)*

WMI event subscriptions are a particularly stealthy persistence mechanism. A WMI subscription can be configured to execute an arbitrary payload in response to system events (e.g., user logon, network connection). This persistence survives reboots and is stored in the WMI repository rather than the filesystem, making it invisible to many file-based detection approaches.

---

## Tactic 6 — Privilege Escalation

NordFinans's AD environment (~3,500 accounts, hybrid AD, CyberArk for ~200 privileged accounts) creates a complex privilege structure with multiple escalation pathways.

---

**T1068 — Exploitation for Privilege Escalation**

The legacy Java components in T24 customizations represent a concrete, named vulnerability surface. Older Java versions carry known privilege escalation vulnerabilities (e.g., deserialization flaws, local privilege escalation via JVM components). On Linux T24 servers, privilege escalation from a T24 application user context to root is achievable through unpatched kernel vulnerabilities. Monthly internal vulnerability scans (Qualys) should catch these, but patch lag — particularly for production core banking servers where maintenance windows are constrained — is a realistic gap.

---

**T1078 — Valid Accounts (Privileged Context)**
*(Sub-technique: T1078.002 Domain Accounts)*

The path from a compromised standard domain account to a privileged account is often shorter than defenders expect. AD misconfigurations (excessive group memberships, Kerberoastable service accounts with weak passwords, ACL over-permissions) are endemic in organizations with 10+ years of AD history. NordFinans's quarterly access recertification covers critical systems, but it does not necessarily audit AD ACLs or nested group memberships — which are common escalation paths.

---

**T1558 — Steal or Forge Kerberos Tickets**
*(Sub-techniques: T1558.003 Kerberoasting, T1558.001 Golden Ticket)*

In hybrid AD environments, Kerberoasting (requesting service tickets for accounts with SPNs and cracking them offline) is a well-documented privilege escalation path. Service accounts in the T24, SWIFT, or SAP S/4HANA integration layer are likely candidates. If an attacker compromises the KRBTGT account hash (requiring domain controller access), a Golden Ticket provides unlimited and persistent Kerberos authentication capability — effectively bypassing CyberArk for on-premises systems. The SWIFT secure zone's reliance on HSMs provides some protection for SWIFT-specific credentials, but the AD plane that governs access to SWIFT workstations remains a target.

---

**T1134 — Access Token Manipulation**
*(Sub-techniques: T1134.001 Token Impersonation/Theft, T1134.003 Make and Impersonate Token)*

On Windows 11 endpoints and Windows Servers, token impersonation allows a process running as a standard user to escalate to SYSTEM or to another user's context. This is particularly relevant in the context of Intune-managed endpoints where the Intune management process (running as SYSTEM) may be exploitable as a token source. CrowdStrike Falcon detects many token manipulation techniques, but novel implementations continue to emerge.

---

**T1484 — Domain or Tenant Policy Modification**
*(Sub-technique: T1484.001 Group Policy Modification)*

An attacker with Domain Admin or Group Policy Creator privileges can modify GPOs to push malicious configurations to all domain-joined endpoints simultaneously — disabling AV, enabling WinRM, or deploying persistent payloads. This is a high-impact escalation-to-lateral-movement technique particularly dangerous in NordFinans's environment where a single AD domain appears to span the Swedish and Nordic branch infrastructure.

---

## Tactic 7 — Defense Evasion

Defense evasion is particularly important given NordFinans's deployed controls: CrowdStrike Falcon EDR, Microsoft Sentinel SIEM, Microsoft Defender for Office 365, and Palo Alto NGFW. Adversaries aware of this stack — which is inferable from job postings and public information — will specifically select evasion techniques effective against it.

---

**T1562 — Impair Defenses**
*(Sub-techniques: T1562.001 Disable or Modify Tools, T1562.002 Disable Windows Event Logging, T1562.006 Indicator Blocking)*

Disabling or disrupting CrowdStrike Falcon is a primary objective for ransomware operators — ALPHV/BlackCat and Akira specifically include CrowdStrike "killer" modules in their toolkits. These typically require local administrator or SYSTEM privileges but can be executed using known driver vulnerabilities (BYOVD — Bring Your Own Vulnerable Driver). Disrupting Sentinel log ingestion by stopping the Microsoft Monitoring Agent or Azure Monitor Agent on compromised hosts would create visibility gaps, especially dangerous given the limited SOC capacity. After-hours attacks timed to exploit the handover between internal SOC and MSSP create natural detection delays.

---

**T1036 — Masquerading**
*(Sub-techniques: T1036.003 Rename System Utilities, T1036.004 Masquerade Task or Service, T1036.005 Match Legitimate Name or Location)*

Renaming malicious tools to match legitimate Windows binaries, naming malicious services to resemble Windows system services, or placing malicious executables in paths where legitimate executables reside (e.g., `C:\Windows\System32\`) are standard LotL evasion techniques. In NordFinans's environment, masquerading payloads as Temenos T24 components or CyberArk processes would exploit the security team's familiarity with these tools as legitimate.

---

**T1027 — Obfuscated Files or Information**
*(Sub-techniques: T1027.002 Software Packing, T1027.010 Command Obfuscation, T1027.013 Encrypted/Encoded File)*

PowerShell obfuscation (encoding commands in Base64, using character substitution, or leveraging the `Invoke-Expression` pattern) is standard tradecraft to evade script-based detection rules in Sentinel. HTML smuggling — delivering encoded payloads that assemble in the browser, bypassing email gateway and proxy inspection — is a common phishing delivery mechanism that has proven effective against Microsoft Defender for Office 365.

---

**T1055 — Process Injection**
*(Sub-techniques: T1055.001 Dynamic-link Library Injection, T1055.012 Process Hollowing)*

Injecting malicious code into legitimate processes (e.g., svchost.exe, Microsoft Teams, or a browser process) causes malicious activity to appear under a trusted process name in both task manager and EDR telemetry. Process hollowing — replacing the memory of a legitimate process with malicious code — is particularly effective against behavioral detection rules that whitelist trusted processes. CrowdStrike Falcon has strong process injection detection, but newer injection techniques targeting edge cases continue to emerge.

---

**T1553 — Subvert Trust Controls**
*(Sub-technique: T1553.002 Code Signing)*

Signing malicious payloads with stolen or fraudulently obtained code signing certificates allows them to pass Windows Defender SmartScreen and many EDR allow-listing checks. For NordFinans, the risk is specifically that internally developed applications signed with the bank's own certificates might not receive the same scrutiny as external software — a compromised Azure DevOps pipeline could produce maliciously modified but correctly signed application packages.

---

**T1564 — Hide Artifacts**
*(Sub-techniques: T1564.001 Hidden Files and Directories, T1564.004 NTFS File Attributes)*

On both Windows and Linux systems, adversaries can hide files and tools using NTFS Alternate Data Streams (Windows) or Linux dotfiles and hidden directories. On the T24 Linux servers, files hidden within the T24 application directory structure would likely only be detected by integrity monitoring tools — which are not mentioned in the profile.

---

## Tactic 8 — Credential Access

Credential access is the gateway to lateral movement and privilege escalation. NordFinans's hybrid AD, mixed MFA enforcement posture, and combination of cloud and on-premises services create a rich credential landscape for adversaries.

---

**T1110 — Brute Force**
*(Sub-techniques: T1110.001 Password Guessing, T1110.003 Password Spraying, T1110.004 Credential Stuffing)*

Password spraying against the Azure AD authentication endpoint is a low-risk (from the attacker's perspective), high-reward technique. By attempting a single common password against many accounts, adversaries avoid account lockout thresholds while iterating through the ~3,500 AD accounts. Credential stuffing against the web banking portal — using lists compiled from unrelated breaches — is a continuous, automated threat affecting retail customers. Azure AD Smart Lockout and Azure AD Identity Protection partially mitigate this, but the profile does not confirm these are fully configured.

---

**T1003 — OS Credential Dumping**
*(Sub-techniques: T1003.001 LSASS Memory, T1003.002 Security Account Manager, T1003.006 DCSync)*

LSASS memory dumping on Windows endpoints and servers (using Mimikatz or similar) extracts plaintext credentials and NTLM hashes cached in memory. CrowdStrike Falcon specifically detects LSASS access attempts and Mimikatz signatures, but kernel-based credential dumping techniques (bypassing user-mode hooks) continue to evade behavioral detection. DCSync — which replicates domain controller credential databases by impersonating a domain controller — requires Domain Admin privileges but would yield the KRBTGT hash and all domain account credentials.

---

**T1558 — Steal or Forge Kerberos Tickets (Credential Access Context)**
*(Sub-technique: T1558.003 Kerberoasting)*

Any AD domain account can request a service ticket for any account with a Service Principal Name (SPN), then crack the encrypted ticket offline. This requires no special privileges and generates minimal log noise. In an environment with T24 integration service accounts, SAP S/4HANA service accounts, and SWIFT operator accounts (all likely to have SPNs), Kerberoasting is a high-probability credential access technique.

---

**T1528 — Steal Application Access Token**

OAuth 2.0 / OpenID Connect tokens issued to TPPs via the Open Banking API gateway represent a novel credential class. An attacker who can steal an OAuth access token — through a compromised TPP, a token leakage vulnerability in the API, or a rogue application registered via a compromised TPP identity — can access customer account information APIs with the full authorization of that TPP. Unlike password-based credentials, OAuth tokens are often not subject to the same monitoring and revocation controls.

---

**T1552 — Unsecured Credentials**
*(Sub-techniques: T1552.001 Credentials in Files, T1552.004 Private Keys, T1552.008 Chat Messages)*

Azure DevOps pipeline configurations, infrastructure-as-code templates, and developer repositories are frequent repositories of inadvertently committed secrets: API keys, service account passwords, Azure storage connection strings, private certificates. Microsoft Teams chat messages (SharePoint-backed) may contain credentials shared informally between administrators ("quick question, what's the password for the T24 batch service account?"). These are non-ephemeral — once committed, they persist in version history even after the credential is nominally changed.

---

**T1539 — Steal Web Session Cookie**

Session cookies for the web banking portal carry significant value. A compromised customer endpoint or a cross-site scripting (XSS) vulnerability in the web application could allow session cookie theft, enabling adversary access to customer accounts with full session privileges — bypassing authentication entirely. Azure Front Door WAF provides XSS protection but is not infallible.

---

## Tactic 9 — Discovery

Post-compromise discovery allows adversaries to map NordFinans's internal environment and identify paths to high-value assets: T24, SWIFT, AD, and customer PII.

---

**T1087 — Account Discovery**
*(Sub-techniques: T1087.001 Local Account, T1087.002 Domain Account, T1087.004 Cloud Account)*

Standard AD enumeration tools (BloodHound, SharpHound, built-in `net` commands) can map all domain accounts, group memberships, and trust relationships within minutes of gaining a domain-joined foothold. Azure AD account discovery via Microsoft Graph API is possible with any authenticated cloud account. This enumeration would quickly reveal the ~200 privileged accounts managed by CyberArk, identifying the highest-value credential targets.

---

**T1069 — Permission Groups Discovery**
*(Sub-techniques: T1069.001 Local Groups, T1069.002 Domain Groups, T1069.003 Cloud Groups)*

BloodHound specifically models AD group relationships to identify the shortest privilege escalation path from a compromised standard account to Domain Admin. In NordFinans's environment, groups granting access to SWIFT workstations, T24 database interfaces, or CyberArk administration are the priority discovery targets. Azure AD role assignments discovery reveals which accounts have Global Administrator, Privileged Role Administrator, or Sentinel Contributor roles.

---

**T1046 — Network Service Discovery**

Internal network scanning (using Nmap, Masscan, or built-in PowerShell) from a compromised host would reveal the internal network segmentation — core banking zone, SWIFT zone, management zone — and identify specific services exposed between zones. Palo Alto NGFW logs inter-zone traffic, so aggressive scanning should trigger alerts, but passive discovery using legitimate Windows networking commands generates lower noise.

---

**T1580 — Cloud Infrastructure Discovery**

Azure subscription enumeration using compromised Azure AD credentials or service principal tokens reveals all deployed resources: Azure VMs, Azure App Services, storage accounts, Key Vaults, Azure API Management instances, and Azure DevOps organizations. This discovery phase is essential for an attacker seeking to move laterally from cloud to on-premises or to identify cloud-hosted PII (e.g., customer data in Azure storage). The profile notes that cloud security controls are "still being standardized," suggesting potential misconfigurations that discovery would reveal.

---

**T1135 — Network Share Discovery**

SMB share enumeration on Windows servers and workstations reveals shared network drives containing financial reports, customer data exports, HR records, and development artifacts. In a bank environment, finance department shared drives often contain highly sensitive data (loan applications, customer PII) that is not covered by the same access controls as primary systems.

---

**T1082 — System Information Discovery**

Gathering OS version, hardware, and configuration information from T24 servers (Linux/Oracle DB) or SWIFT Alliance Lite2 infrastructure is a precursor to exploitation. This discovery confirms whether specific vulnerabilities (e.g., unpatched Java CVEs, Oracle DB vulnerabilities) are exploitable on the target system.

---

## Tactic 10 — Lateral Movement

NordFinans's hybrid network — with MPLS-connected branches, VPN-connected vendor systems, cloud-to-on-premises connectivity, and an active-passive data center pair — creates multiple lateral movement pathways.

---

**T1021 — Remote Services**
*(Sub-techniques: T1021.001 Remote Desktop Protocol, T1021.002 SMB/Windows Admin Shares, T1021.004 SSH, T1021.006 Windows Remote Management)*

RDP is the primary lateral movement vector across Windows endpoints and servers. NTLM relay attacks using SMB allow credential forwarding from a compromised host to gain access to other Windows systems without the original password. SSH lateral movement applies to the RHEL servers running T24 components. WinRM (Windows Remote Management) enables PowerShell Remoting — an administrative tool that is functionally equivalent to remote code execution once credentials are available.

---

**T1550 — Use Alternate Authentication Material**
*(Sub-techniques: T1550.002 Pass the Hash, T1550.003 Pass the Ticket)*

Pass-the-Hash and Pass-the-Ticket attacks allow an attacker who has harvested NTLM hashes or Kerberos tickets (via T1003 or T1558) to authenticate to remote systems without knowing the plaintext password. In NordFinans's environment, a NTLM hash obtained from a compromised workstation could be passed to authenticate to a file server, then to a database server, then ultimately to a T24 management interface — all without any additional credential compromise. Palo Alto NGFW does not inspect NTLM or Kerberos at the credential level; this lateral movement would appear as legitimate administrative traffic.

---

**T1563 — Remote Service Session Hijacking**
*(Sub-technique: T1563.002 RDP Hijacking)*

Active RDP sessions on Windows Servers can be hijacked by an attacker running as SYSTEM, allowing them to take over a logged-in administrator's session — inheriting all open applications, tokens, and connections. This is particularly dangerous if a CyberArk-vaulted privileged session is active on a target server.

---

**T1534 — Internal Spearphishing**

Once inside the M365 environment, an adversary can send phishing emails from a compromised internal account, which bypass external email filtering. Internal emails from a known colleague are far more likely to be trusted. A compromised finance department account could send a convincing email to SWIFT operators requesting action on a fraudulent transaction or credential, leveraging internal trust relationships.

---

**T1570 — Lateral Tool Transfer**

Moving hacking tools and payloads between compromised systems within the network — using SMB, RDP clipboard, or legitimate file transfer utilities (certutil, BITSAdmin, curl) — is a precursor to most lateral movement operations. The T24 and SWIFT secure zone isolation is only effective if inter-zone tool transfer is also blocked; firewall rules controlling file transfer protocols between zones are a critical control.

---

## Tactic 11 — Collection

Collection targets NordFinans's highest-value data assets: customer financial records, SWIFT transaction data, PII across four jurisdictions, and internal M365 communications.

---

**T1114 — Email Collection**
*(Sub-techniques: T1114.001 Local Email Collection, T1114.002 Remote Email Collection, T1114.003 Email Forwarding Rule)*

M365 Exchange Online is a treasure trove for adversaries: board-level communications, M&A discussions, regulatory correspondence, and operational credentials shared via email. Email forwarding rules — silently configured via Outlook Web Access or Exchange PowerShell — allow continuous exfiltration of all incoming and outgoing emails to an external address without generating ongoing access events. Microsoft Defender for Office 365 and Microsoft Sentinel should detect anomalous forwarding rule creation, but this relies on alert rules being properly configured and actively monitored.

---

**T1213 — Data from Information Repositories**
*(Sub-techniques: T1213.002 Sharepoint, T1213.003 Code Repositories)*

SharePoint Online (via M365) and Microsoft Teams host NordFinans's internal documents: credit risk models, internal audit reports, SWIFT operating procedures, customer data exports, and HR records. An adversary with a compromised M365 account can browse and download SharePoint content at scale. Azure DevOps repositories contain source code, infrastructure-as-code, pipeline definitions, and potentially hard-coded secrets — all high-value intelligence for deepening an attack.

---

**T1530 — Data from Cloud Storage**

Azure Blob Storage used by the web banking backend microservices may store customer document uploads, transaction history exports, or application logs containing PII. Misconfigured Azure storage accounts (e.g., public blob containers, overly permissive SAS tokens) are a systemic cloud risk, particularly relevant given that cloud security controls are "still being standardized."

---

**T1005 — Data from Local System**

On the T24 core banking servers, an adversary with root or Oracle DBA access can directly query the Oracle database for customer account records, transaction histories, and financial data. This is the highest-impact data collection scenario — a direct dump of the core banking database. The quarterly access recertification for core banking helps limit the number of accounts with DBA-level privileges, but an attacker who has escalated to root on a T24 server can bypass application-level controls.

---

**T1074 — Data Staged**
*(Sub-technique: T1074.002 Remote Data Staging)*

Before exfiltration, collected data is typically staged in a central location within the compromised environment — compressed and encrypted to minimize exfiltration size and avoid content inspection. Azure Blob Storage (using a compromised service principal) or a staging server in the development network zone (which may have fewer egress controls than production) are likely staging locations.

---

**T1185 — Browser Session Hijacking**

Man-in-the-browser attacks — malicious browser extensions or injected JavaScript — targeting NordFinans's internal banking operations staff could capture transaction data, customer records, or authorization credentials in real time as they appear in the web browser. This is particularly relevant for internal staff accessing T24 via web-based administrative interfaces.

---

## Tactic 12 — Command and Control

NordFinans's outbound traffic profile — heavy Azure traffic, M365 communications, SWIFT messaging, and development pipeline traffic — provides multiple channels for C2 communication that blend into legitimate activity.

---

**T1071 — Application Layer Protocol**
*(Sub-techniques: T1071.001 Web Protocols, T1071.003 Mail Protocols, T1071.004 DNS)*

HTTPS C2 over port 443 to Azure-hosted infrastructure (or to domains hosted on Azure CDN) is essentially indistinguishable from NordFinans's legitimate Azure traffic at the network level. Palo Alto NGFW SSL/TLS inspection is a partial mitigation, but certificate pinning by malware and the sheer volume of Azure traffic make inspection challenging. DNS-based C2 — encoding data in DNS queries to a controlled domain — is extremely difficult to detect without dedicated DNS analytics, which the profile does not mention. SMTP/IMAP-based C2 via M365 (exchanging C2 commands via email drafts) blends into normal mail traffic.

---

**T1572 — Protocol Tunneling**

Tunneling C2 traffic over legitimate protocols (DNS, ICMP, HTTP/2, WebSockets) or through legitimate cloud services (Azure Service Bus, Microsoft Graph API) is a proven technique for evading deep packet inspection. In NordFinans's environment, C2 traffic tunneled over the Microsoft Teams WebSocket connection or over Azure Service Bus messaging would be nearly impossible to distinguish from legitimate application traffic without purpose-built behavioral analytics.

---

**T1090 — Proxy**
*(Sub-techniques: T1090.002 External Proxy, T1090.003 Multi-hop Proxy)*

Multi-hop proxy chains (adversary → Tor → VPS → NordFinans victim) obscure the true C2 origin, complicating attribution and blocking. An adversary may use compromised infrastructure in neutral jurisdictions (or within NordFinans's own Azure tenant) as proxy nodes, ensuring C2 traffic appears to originate from trusted cloud ranges.

---

**T1568 — Dynamic Resolution**
*(Sub-technique: T1568.002 Domain Generation Algorithms)*

DGA-based malware generates pseudo-random C2 domain names algorithmically, making domain-based blocking ineffective. If NordFinans's network does not perform deep DNS behavioral analytics (e.g., detecting unusually high volumes of NXDOMAIN responses characteristic of DGA), DGA-based C2 would operate undetected until the malware's seed is identified.

---

**T1008 — Fallback Channels**

Sophisticated implants maintain multiple C2 channels with failover capability. If the primary HTTPS C2 channel is blocked by Palo Alto NGFW or disrupted by incident response actions, the implant falls back to an alternate channel (DNS, ICMP, or a different cloud service). This resilience makes C2 disruption difficult during active incident response.

---

## Tactic 13 — Exfiltration

NordFinans's regulatory obligations (GDPR, DORA incident reporting) make data exfiltration a high-consequence event. The organization's heavy use of Azure creates both exfiltration risks and detection opportunities.

---

**T1048 — Exfiltration Over Alternative Protocol**
*(Sub-techniques: T1048.001 Exfiltration Over Symmetric Encrypted Non-C2 Protocol, T1048.002 Exfiltration Over Asymmetric Encrypted Non-C2 Protocol)*

Exfiltrating data via SFTP, FTPS, or encrypted custom protocols that are not inspected at the perimeter is a common technique. If Palo Alto NGFW does not perform egress filtering for all non-standard protocols (beyond blocking known-bad IPs), bulk data exfiltration over non-HTTP protocols may succeed.

---

**T1567 — Exfiltration to Cloud Storage**
*(Sub-techniques: T1567.002 Exfiltration to Cloud Storage, T1567.004 Exfiltration to Code Repository)*

Exfiltrating data to legitimate cloud storage services — OneDrive (via M365 credentials), Azure Blob Storage (via a compromised service principal), or GitHub/Azure Repos (via a developer account) — is difficult to distinguish from legitimate data transfers. OneDrive sync from a compromised endpoint will appear as normal M365 activity. Microsoft Defender for Cloud Apps (if deployed) can detect anomalous bulk upload behavior, but the profile does not confirm this control is in place.

---

**T1041 — Exfiltration Over C2 Channel**

For moderate volumes of data (e.g., compressed credential dumps, configuration files, SWIFT transaction logs), exfiltrating over the established HTTPS C2 channel is operationally simple and avoids creating additional outbound connections that might be anomalous.

---

**T1029 — Scheduled Transfer**

Transferring data in small increments on a scheduled basis (e.g., daily at 03:00 CET) avoids triggering volume-based DLP or bandwidth anomaly alerts. The overnight timing coincides with T24 batch processing windows and the reduced SOC monitoring capacity, making detection less likely.

---

**T1030 — Data Transfer Size Limits**

Deliberately limiting individual transfer sizes to stay below DLP thresholds (if DLP tools monitor for large single transfers) is a standard evasion technique. If Sentinel DLP rules alert on transfers over 100MB, an adversary will transfer 99MB at a time repeatedly.

---

## Tactic 14 — Impact

Impact techniques represent the adversary's ultimate objectives. For NordFinans, three distinct impact scenarios are relevant: financial theft (SWIFT fraud), ransomware, and regulatory/reputational damage through data destruction or disclosure.

---

**T1657 — Financial Theft**

SWIFT-targeting actors — historically FIN7, Lazarus Group, and other APT actors with a track record of SWIFT fraud (Bangladesh Bank 2016, FASTCash operations) — represent the highest-financial-impact threat. An adversary who gains access to the SWIFT Alliance Lite2 infrastructure could submit fraudulent SWIFT MT103 payment orders, potentially transferring hundreds of millions of euros before detection. NordFinans's SWIFT CSP compliance (including HSMs, dedicated SWIFT secure zone, and CyberArk-vaulted SWIFT operator credentials) significantly raises the barrier, but the SOC's business-hours-only internal coverage means a Friday evening attack could persist for 14+ hours before the internal team engages.

---

**T1486 — Data Encrypted for Impact (Ransomware)**

Ransomware is the highest-probability, high-impact threat for organizations of NordFinans's profile. The Akira, LockBit 3.0, and BlackBasta ransomware groups have actively targeted Nordic financial institutions. Ransomware targeting the ~2,200 Windows endpoints and ~150 on-premises servers would disrupt retail banking operations. Ransomware reaching the T24 Oracle DB environment could encrypt database files, rendering core banking inoperative. The 4-hour RTO for core banking restoration (backup infrastructure "partially air-gapped") is achievable only if backups are confirmed clean and the ransomware is fully contained — both of which are uncertain during an active incident.

---

**T1490 — Inhibit System Recovery**

Before deploying ransomware, sophisticated operators delete Volume Shadow Copies (VSS), disable Windows Backup, and corrupt or destroy accessible backup repositories. The profile notes that backup infrastructure is "partially air-gapped (tape storage offsite)" — the partially qualifier is significant. If the network-accessible portion of backup infrastructure (daily incremental and weekly full backups) can be reached from compromised servers, backup destruction can precede encryption, transforming a recoverable incident into a catastrophic data loss event. DORA compliance requirements for ICT business continuity make backup resilience a regulatory obligation as well as an operational one.

---

**T1485 — Data Destruction**

Nation-state actors with destructive intent (e.g., linked to geopolitical tensions — relevant for a Swedish bank given Sweden's NATO accession and the geopolitical environment) may prioritize data destruction over financial gain. Wiping T24 database files, destroying Oracle redo logs, and corrupting T24 application code would cause sustained operational disruption requiring full system restoration from backups. Wiper malware (e.g., HermeticWiper, WhisperGate variants) has been used against financial infrastructure in geopolitical conflict contexts.

---

**T1561 — Disk Wipe**
*(Sub-techniques: T1561.001 Disk Content Wipe, T1561.002 Disk Structure Wipe)*

Master Boot Record (MBR) wiping on Windows Servers renders systems unbootable without full OS restoration, compounding the recovery timeline beyond the 4-hour RTO. On Linux T24 servers, filesystem-level wipes of the Oracle data directories achieve equivalent disruption.

---

**T1489 — Service Stop**

Stopping critical services — T24 application servers, Oracle database listeners, SWIFT Alliance Lite2 messaging agents, or the Azure AD Connect synchronization service — can cause targeted disruption without encryption or destruction. Stopping the AD Connect service severs synchronization between on-premises AD and Azure AD, potentially locking cloud service access for all staff simultaneously. This could be used as a coercive technique in ransomware extortion or as a diversion during a simultaneous SWIFT fraud operation.

---

**T1491 — Defacement**
*(Sub-technique: T1491.002 External Defacement)*

Defacement of the web banking portal — replacing it with politically motivated content, disinformation, or extortion messages — causes immediate reputational damage and customer panic, potentially triggering a bank run scenario. For a publicly traded bank on Nasdaq Stockholm, web defacement causing customer withdrawal behavior is a market-impacting event. Azure Front Door WAF provides some protection, but a compromise of the Azure App Service hosting the portal (rather than the WAF layer) could enable defacement.

---

**T1659 — Content Injection**

Injecting fraudulent content into the web banking or mobile banking interface — displaying false balances, fraudulent transaction confirmations, or fake payment instructions — is a high-sophistication attack that manipulates customer behavior without necessarily compromising the underlying T24 database. A compromised Azure API Management gateway or a rogue microservice injected through the CI/CD pipeline could intercept and modify API responses in transit, presenting falsified financial data to customers.
